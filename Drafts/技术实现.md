# 技术实现

## 整体结构

整体是一个API，

输入：
1. 原始视频文件的URL
2. 原始语种
3. 结果语种

输出：
1. 结果视频文件的URL

API使用Python的FastAPI框架。

有任务队列，使用 celery + redis 。

有任务状态数据库，使用 MySQL ，数据表中记录任务id和任务状态。

## API 运行流程

1. API 接到客户端发来的请求，验证参数是否正确，如错误则返回错误信息。
2. 后端将任务加入队列尾部。
3. 后端将任务id+任务状态+接收时间insert入MySQL中，任务状态为排队中。
4. 返回客户端已接收到任务的消息。

## worker 运行流程

1. worker取出队列中最早的一条任务，开始处理。
2. worker处理任务的过程中，每一步开始及结束时，更新MySQL中的任务状态，直至该任务处理结束，任务状态中更新为结果视频的URL。如果中途遇到错误中断，则将错误信息更新到任务状态。
3. 重回第1步，无限循环。

## 前端运行流程

1. 发送请求时，以接到返回信息为止。
2. 查看进度：通过前端页面查MySQL数据。
3. 查看结果：任务完成后，得到结果视频的URL。

## worker 内部逻辑

1. 函数 fetch_video ：通过原始视频URL获取视频文件 source_video 。
2. 函数 fetch_audio ：获取视频文件中的音频，保存成音频文件 source_audio 。
3. 函数 separate_audio ：将音频文件分离成两个音频文件，一个是语音（speech_audio），一个是其他声音（noise_audio）。
4. 函数 asr ：通过外部 ASR API 将 speech 文件转化为文本，文本中按句间断，每一句都配有起始时间和结束时间（精确到ms）。
5. 建 sentences 列表，列表中将有序记录每一个句子，每个句子有属性：句子起始时间 sentence_start_time、句子结束时间 sentence_end_time、句子时长 sentence_duration、句子原始音频 sentence_source_audio、句子原文 sentence_source_text、句子译文 sentence_target_text、句子译后TTS音频 sentence_target_audio。
6. 函数 cut_speech ：根据ASR结果中每一句的起始时间和结束时间，切割语音音频 speech_audio ，生成若干个句子音频文件，句子与句子中间的间隔部分丢弃，并将切割后的句子音频 sentence_source_audio、句子起始时间 sentence_start_time、句子结束时间sentence_end_time、句子时长（根据起止时间计算得出）sentence_duration 存入sentences列表，中每句的对应属性中。
7. 函数 translate_sentences ：调用外部 LLM API ，翻译sentences[]中每个句子，prompt中要带有asr后的原文全文，带每一句的时长，让 LLM 注意翻译的结果长度要基本满足时长要求，以免在TTS后与原句时长相差太多。将译后的每一句存入sentences列表中每一句的 sentence_target_text 。
8. 函数 tts ：调用外部的 index-tts-2 模型，输入每一句的 sentence_source_audio （用于模仿音色及情绪）、sentence_target_text，生成出对应的 sentence_target_audio ，并存入sentences[]。
9. 函数 refine_target_audio ：提取每一个sentence_target_audio的时长，与其sentence_duration对比，如果时长介于sentence_suration的0.85倍-1.15倍之间，则使用ffmpeg将sentence_target_audio拉或缩到与sentence_duration等长，更新sentence_target_audio。如果超出0.85倍-1.15倍的范围，则调用外部 LLM API ，重新翻译，使之能够适应应有的时长，翻译后再次 tts 并再次拉/缩时长，直至所有的 sentence_target_audio 时长都等于 sentence_duration 。
10. 函数 assemble_target_video ：使用 ffmpeg 将静音后的 source_video 、noise_audio 、 sentence_target_audio 进行组装，每一个 sentence_target_audio 根据 sentence_start_time 放置，最终生成结果视频 target_video 。
