# MagicDub Brain

这是 MagicDub 项目的 **`*-brain` 仓库**。它不是代码仓库，而是项目的 **「思考 — 沉淀 — 交付」系统**：把零散灵感、外部信息、与 AI 的对话，逐步整理成可执行的设计与文档，并作为项目落地实现的 **唯一事实来源（Single Source of Truth）**。

> 范围约束：本项目 **永远不做唇形/口型修正**，仅追求「译制片式」音频替换与时间轴对齐体验。

---

## 这个仓库是给谁看的？

- **主要给 AI agent 看**：让任何一个 agent 在介入项目时，能快速、全面地了解项目背景、目标、决策与现状，从而更好地协助你。
- **人类重点看 `Releases/`**：定稿内容集中在这里。`Drafts/` 与 `Archive/` 主要用于补充上下文和历史脉络。

如果你是一个刚接入本项目的 AI agent，请先阅读 [`AGENTS.md`](./AGENTS.md)。

---

## 目录结构

```
magicdub-brain/
├── README.md          # 项目入口与说明
├── AGENTS.md          # 给 AI agent 的操作指南（最重要）
├── Releases/          # 唯一事实来源：定稿文档
├── Drafts/            # 草稿区：灵感、外部信息、与 AI 的对话
└── Archive/           # 归档区：已完成使命或被弃用内容
```

| 目录 | 定位 | 放什么 | 原则 |
| --- | --- | --- | --- |
| **`Releases/`** | 唯一事实来源 | 定稿的项目说明、设计方案、开发文档、规范、路线图 | 默认「可照着做」，不写不确定的猜想 |
| **`Drafts/`** | 草稿 / AI 上下文素材 | 零碎想法、灵感、外部资料、与 AI 的对话记录、半成品梳理 | 允许混乱，但写清楚「为什么写它、卡在哪、下一步要补什么」 |
| **`Archive/`** | 历史归档 | 已完成使命或不再适用的文档 | 不删，但日常执行以 `Releases/` 为准 |

---

## 推荐工作流（从 0 到 1）

1. **先丢进 `Drafts/`** —— 任何灵感、需求片段、外部资料、AI 对话先快速记录，不求完美。
2. **持续打磨草稿** —— 每次迭代都让它更「可执行」：补流程、接口、示例、边界条件、风险、验证方式。
3. **达到可交付标准后移入 `Releases/`** —— 标准是：读者（人或 AI）不用追问也能照做。
4. **被替代或过时后移入 `Archive/`** —— 并在 `Releases/` 新文档里写明替代关系 / 迁移指引。
